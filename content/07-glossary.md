# Glossary

## Common Terms

### A
**API** - Application Programming Interface; a set of rules for software communication  
**Anomaly** - Unusual or unexpected data pattern requiring investigation  
**Audit** - Systematic review of system access and data changes

### B
**BigMile** - Fleet management system used for tracking vehicles and operations  
**Business Owner** - Executive responsible for a specific business function or department

### C
**CBS** - Central Business System; core ERP system for financial and operational data  
**Customer Master** - Central database of customer information and attributes  
**CSV** - Comma-separated values; a data file format

### D
**Dashboard** - Visual display of key metrics and reports  
**Data Warehouse** - Centralized repository of business data  
**Drill-Down** - The ability to explore more detailed data from a summary view  
**DQI** - Data Quality Index; measure of data accuracy and completeness

### E
**ETL** - Extract, Transform, Load; the process of moving data between systems  
**Export** - Download or extract data from a report

### F
**Financial Operations** - Reports related to accounting and financial management  
**Fleet Hours** - Reports on vehicle usage and driver hours  
**Fuel Reporting** - Analytics on fuel consumption and costs

### G
**GL** - General Ledger; the main accounting record

### H
**Hash** - Security function used to protect sensitive data

### I
**Invoice** - Document requesting payment for goods or services

### K
**KPI** - Key Performance Indicator; important business metric

### L
**Load** - Process of moving data into the data warehouse

### M
**Maintenance** - Planned system updates or repairs  
**Metadata** - Data that describes other data

### O
**Owner** - Team member responsible for a specific report or dataset

### P
**Power BI** - Business analytics and visualization tool used for reports  
**Permissions** - Access rights granted to users

### R
**Refresh** - Update of data in reports from source systems  
**Report Owner** - Person responsible for accuracy and maintenance of a report  
**Role** - User type that determines system access level

### S
**Schedule** - Automated delivery of reports to users  
**SLA** - Service Level Agreement; commitment to system performance  
**Source System** - Original system where data is created

### T
**Transform** - Process of cleaning and organizing data for analysis

### U
**User** - Individual accessing the reporting system

### V
**Visualization** - Graphical representation of data (charts, graphs, etc.)

### W
**Webhook** - Automated message sent when an event occurs
