# Access & Permissions

## Overview

This document describes user roles, access levels, and how to request or modify permissions in the AB Texel reporting system.

## User Roles

### Executive
- **Access Level:** Full system access
- **Reports:** All dashboards and reports
- **Data:** Full organizational view
- **Permissions:** View, export, schedule reports
- **Use Case:** C-level and senior management

### Manager
- **Access Level:** Departmental + related data
- **Reports:** Department dashboards + shared reports
- **Data:** Department and team-level views
- **Permissions:** View, export, filtered drill-down
- **Use Case:** Team leads and department managers

### Analyst
- **Access Level:** Full technical access
- **Reports:** All reports + development environment
- **Data:** Full access with detailed drill-down
- **Permissions:** View, export, create views, troubleshoot
- **Use Case:** Data analysts and report developers

### Viewer
- **Access Level:** Limited reports only
- **Reports:** Assigned reports only
- **Data:** Summary level only
- **Permissions:** View, export
- **Use Case:** General staff and clients

### Read-Only
- **Access Level:** View-only
- **Reports:** Specific assigned reports
- **Data:** Summary level, no drill-down
- **Permissions:** View only (no export)
- **Use Case:** External stakeholders and auditors

## Permission Levels

### Report Permissions
- **View** - Can access and read reports
- **Export** - Can download data (CSV, Excel)
- **Schedule** - Can set up automated deliveries
- **Share** - Can share with other users
- **Edit** - Can modify report design

### Data Permissions
- **View** - Can see data in reports
- **Drill-Down** - Can explore detailed data
- **Export** - Can export underlying data
- **None** - No access to data

## Requesting Access

To request or modify permissions:
1. Submit a request through the access portal
2. Specify role needed and business justification
3. Get manager approval
4. Access is granted within 1-2 business days

See [Contact & Support](10-contact-support.md) for submission instructions.

## Security & Confidentiality

- All access is logged and audited
- Data is encrypted in transit and at rest
- Sensitive information requires additional approval
- Permissions are reviewed quarterly
